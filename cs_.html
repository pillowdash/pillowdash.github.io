Section 6: Modularization

Key Concepts

  • Modules consist of interface (.h) and implementation (.c) files

  • Advantages: reusability, maintainability, abstraction

  • Scope: program scope vs module scope (using static)

  • Global variables in modules use extern in header files

  • Opaque structures hide implementation details

Practice Questions

Q1: What are the three main advantages of modularization?
A1: Reusability (same module in many projects), maintainability (easier to debug/test single modules), and abstraction (client only needs to understand interface, not implementation).

Q2: How would you declare a global constant in a module that should be accessible to clients?

// In .h file:
extern const int MAX_SIZE;

// In .c file:

Q3: What's wrong with this module interface/implementation?

c
Copy
Download
// prime.h
bool is_prime(int n);

// prime.c
static bool is_prime(int n) {
    // implementation
}

A3: The function is declared as static in the implementation which makes it module-scoped, conflicting with the header declaration that makes it program-scoped. Remove static from the implementation.

// rational.c

#include <stdio.h>

void print_rational(const struct rational *r) {
    printf("%d/%d", r->numerator, r->denominator);
}

struct rational rational_add(struct rational a, struct rational b) {
    struct rational result = {
        a.numerator * b.denominator + b.numerator * a.denominator,
        a.denominator * b.denominator
    };
    return result;
}

Section 7: Efficiency

Key Concepts

  • Big-O notation describes algorithm growth rates

  • Common time complexities: O(1), O(log n), O(n), O(n log n), O(n²), O(2ⁿ)

  • Analyzing loops (nested loops multiply complexities)

  • Recurrence relations for recursive functions

  • Sorting algorithms and their complexities

Practice Questions

Q1: What is the time complexity of this function?

c
Copy
Download
int sum_matrix(int mat[][N], int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < N; j++) {
            sum += mat[i][j];
        }
    }
    return sum;
}

A1: O(n) - The inner loop runs N times (constant), so we have O(n × N) = O(n) since N is constant.

Q2: Solve the recurrence relation: T(n) = 2T(n/2) + O(n)
A2: This is the merge sort recurrence, which solves to O(n log n) using the Master Theorem.

Q3: Why is quicksort's worst case O(n²) but it's still commonly used?
A3: Worst case occurs when pivot is smallest/largest element, but average case is O(n log n) and it has good cache performance. Pivot selection strategies can minimize worst case.

Coding Challenge

Implement Binary Search:

c
Copy
Download
int binary_search(const int arr[], int len, int target) {
    int left = 0;
    int right = len - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}

Section 8: Dynamic Memory & ADTs

Key Concepts

  • Heap memory management with malloc/free

  • Memory leaks and dangling pointers

  • Resizing arrays with realloc

  • Amortized analysis (doubling strategy)

  • Opaque structures for ADTs

Practice Questions

Q1: What's wrong with this code?

c
Copy
Download
int *create_array(int n) {
    int arr[n];
    return arr;
}

A1: Returning a pointer to a stack-allocated array. The array will be destroyed when the function returns. Should use malloc for heap allocation.

Q2: How does the doubling strategy for dynamic arrays achieve O(1) amortized time for append operations?
A2: By doubling capacity when full, the occasional expensive resize is "amortized" over many cheap inserts, making average cost constant.

Q3: What are the advantages of opaque structures?
A3: Better encapsulation, prevents clients from directly modifying internal state, allows implementation changes without affecting clients.

Coding Challenge

Implement a Stack ADT using Dynamic Arrays:

c
Copy
Download
// stack.h
struct stack;
struct stack *stack_create();
void stack_destroy(struct stack *s);
void stack_push(struct stack *s, int value);
int stack_pop(struct stack *s);
int stack_top(const struct stack *s);
bool stack_is_empty(const struct stack *s);

// stack.c
struct stack {
    int *data;
    int size;
    int capacity;
};

struct stack *stack_create() {
    struct stack *s = malloc(sizeof(struct stack));
    s->capacity = 1;
    s->size = 0;
    s->data = malloc(s->capacity * sizeof(int));
    return s;
}

void stack_push(struct stack *s, int value) {
    if (s->size == s->capacity) {
        s->capacity *= 2;
        s->data = realloc(s->data, s->capacity * sizeof(int));
    }
    s->data[s->size++] = value;
}

Section 9: Linked Data Structures

Key Concepts

  • Linked list node structure (data + next pointer)

  • Wrapper structures for cleaner interfaces

  • Recursive vs iterative traversal

  • Augmentations (like tail pointers)

  • Binary search trees and their operations

Practice Questions

Q1: What's the time complexity of inserting at the front/back of a linked list?
A1: Front: O(1), Back: O(n) without tail pointer, O(1) with tail pointer.

Q2: How would you detect a cycle in a linked list?
A2: Use Floyd's cycle-finding algorithm (tortoise and hare with two pointers moving at different speeds).

Q3: What makes a BST unbalanced and why is this bad?
A3: When nodes are inserted in sorted order, creating a linked-list-like structure with height O(n) instead of O(log n), making operations O(n).

    int data;
    struct llnode *next;
};

struct llist {
    struct llnode *head;
    struct llnode *tail;
};

void list_add_back(struct llist *list, int value) {
    struct llnode *node = malloc(sizeof(struct llnode));
    node->data = value;
    node->next = NULL;
    
    if (list->tail) {
        list->tail->next = node;
    } else {
        list->head = node;
    }
    list->tail = node;
}

Section 10: Generic ADTs

Key Concepts

  • Void pointers for generic data

  • Generic comparison functions

  • Function pointers for customization

  • Memory management in generic ADTs

  • Tradeoffs in data structure selection

Practice Questions

Q1: How does qsort handle comparison of different types?
A1: It uses a function pointer to a comparison function that takes void pointers and returns negative/0/positive.

Q2: What's the purpose of the size parameter in qsort?
A2: It specifies the size of each element in bytes, allowing qsort to correctly calculate memory offsets.

Q3: When designing a generic ADT, what are some considerations for memory management?
A3: Need to decide who owns/frees memory - ADT or client. May need to provide free functions. For complex types, may need copy functions.

Generic Stack ADT:

Copy
Download
// gstack.h
struct gstack;
struct gstack *gstack_create(void (*free_func)(void *));
void gstack_destroy(struct gstack *s);
void gstack_push(struct gstack *s, void *data);
void *gstack_pop(struct gstack *s);

// gstack.c
struct gstack {
    void **data;
    int size;
    int capacity;
    void (*free_func)(void *);
};

void gstack_destroy(struct gstack *s) {
    if (s->free_func) {
        for (int i = 0; i < s->size; i++) {
            s->free_func(s->data[i]);
        }
    }
    free(s->data);
    free(s);
}
