1. Dynamic-array Allocation Strategy

Question:
What strategy improves average run-time when dynamically allocating an array of unknown length?
  • a. Wrapper
  • b. Functional
  • c. Augmentation
  • d. Doubling
Answer:
d. Doubling Strategy
  • Doubling the capacity when full keeps the amortized cost per insertion O(1), because the expensive O(n) resize happens only every n/2 steps.

2. C Strings vs. Char Arrays

Question:
Which statement is true?
  • a. C strings always live in read-only memory.
  • b. Arrays cannot be initialized with "...".
  • c. Array stores its length.
  • d. C strings are null-terminated.
  • e. No difference.
Answer:
d. C strings are null-terminated.
  • The null byte '\0' marks the end; arrays can be initialized with "..." and can reside anywhere (stack, heap, read-only). Arrays do not store length.

3. Linked-list Add-to-back Technique

Question:
To add efficiently to the back, we should:
  • a. Augmentation
  • b. Recursion in add_to_back
  • c. Iteration in add_to_back
  • d. Wrapper
Answer:
a. Augmentation
  • Storing a tail pointer inside the list (augmenting the node) makes insertion O(1).

4. BST Height in O(1)

Question:
How can we obtain height in O(1)?
  • a. Augment node with subtree height.

a. Augment node with subtree height

bool bst_search(struct bstnode *bst, int i) {
    if (!bst) return false;
    if (i == bst->key) return true;
    return i < bst->key ? bst_search(bst->left, i) : bst_search(bst->right, i);
}
(b) Iterative
c
Copy
bool bst_search(struct bstnode *bst, int i) {
    while (bst) {
        if (i == bst->key) return true;
        bst = (i < bst->key) ? bst->left : bst->right;
    }
    return false;
}

17. Filter Array

Copy
int filter(int **arr, int n, bool (*pred)(int)) {
    int write = 0;
    for (int read = 0; read < n; ++read) {
        if (pred((*arr)[read])) {
            (*arr)[write++] = (*arr)[read];
        }
    }
    int *tmp = realloc(*arr, write * sizeof(int));
    if (tmp) *arr = tmp;
    return write;
}

19. Robot ADT

Header & Implementation Sketch
c
Copy
typedef struct instruction {
    char cmd;
    struct instruction *next;
} Instr;

typedef struct robot {
    int x, y;
    Instr *head, *tail;
} *Robot;

Robot create_Robot(void) {
    Robot r = malloc(sizeof(struct robot));
    r->x = r->y = 0;
    r->head = r->tail = NULL;
    return r;
}

void store_instruction(Robot r, char c) {
    Instr *n = malloc(sizeof(Instr));
    n->cmd = c; n->next = NULL;
    if (!r->head) r->head = r->tail = n;
    else { r->tail->next = n; r->tail = n; }
}

void go(Robot r) {
    for (Instr *p = r->head; p; p = p->next) {
        switch(p->cmd) {
            case 'u': r->y++; break;
            case 'd': r->y--; break;
            case 'l': r->x--; break;
            case 'r': r->x++; break;
        }
        printf("(%d,%d)\n", r->x, r->y);
    }
}

void destroy_Robot(Robot r) {
    Instr *p = r->head;
    while (p) { Instr *n = p->next; free(p); p = n; }
    free(r);
}

1. Advantages of Modularization

Q: Give three advantages of using modularization and describe each.
A:
  • Reusability: A module can be reused in many programs without rewriting code.
  • Maintainability: Bugs are localized; fixing one module does not disturb others.
  • Abstraction: Clients only need the interface, hiding implementation details.

2. True / False about Modules

Q: Which statement(s) are true?
a. Clients may require functions from modules.
b. Clients provide implementation to modules.
c. Modules provide functions to clients.
A:
  • a. True – clients call functions that modules export.
  • b. False – modules provide the implementation; clients only use it.
  • c. True – modules expose functions to clients.

3. Compilation & Linking

Q: Which statements are true?
a. .o files are compiled machine code.
b. Only one module is allowed per program.
c. Multiple .o files can be combined.
d. A program must have exactly one function called main.
A:
  • a. True – .o is object code.
  • b. False – many modules can coexist.
  • c. True – the linker combines them.
  • d. True – exactly one main is required.

4. Declaration vs Definition

Q: Discuss declaration vs definition.
A:
  • Declaration tells the compiler “this identifier exists somewhere” (type + name).
  • Definition actually allocates storage or provides the body of a function.

5. extern Keyword

Q: Describe extern.
A:
extern declares a variable or function defined in another translation unit. The linker resolves the actual definition at link-time.

6. Header Inclusion

Q: True / False: my_module.c will not compile unless you include my_module.h.
A: False – compilation only needs declarations; the header is conventional, not mandatory.

8. Opaque Structure

Q: Describe an opaque structure.
A: An opaque structure is declared via a forward struct pointer whose full definition is hidden from the client (only visible in the implementation file).

    for (int i = 0; i < n; ++i) freq[(unsigned char)s[i]]++;
