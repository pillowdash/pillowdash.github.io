Questions & Answers

Q1: Why do we need function declarations in C for mutual recursion?
A1: The C compiler processes code top-down. When it encounters a call to a not-yet-defined function, it needs a declaration to know the function's signature (return type and parameters). Without declarations, the compiler would make incorrect assumptions about undeclared functions.

Q2: What are the three main advantages of modularization?
A2:

  1. Reusability - modules can be used in multiple projects

  2. Maintainability - easier to test/debug individual modules

  3. Abstraction - clients only need to understand the interface, not implementation

Q3: What's the difference between a transparent and opaque structure in modules?
A3:

  • Transparent: Structure definition is in .h file, clients can access fields directly

  • Opaque: Only forward declaration in .h, actual definition in .c file, clients must use interface functions

Questions & Answers

Q1: What is the time complexity of this function?

c
Copy
Download
int sum_array(const int a[], int len) {
    int sum = 0;
    for (int i = 0; i < len; ++i) {
        sum += a[i];
    }
    return sum;
}

A1: O(n) where n is the array length. Each element is visited exactly once.

Q2: Solve the recurrence relation: T(n) = O(1) + T(n-1)
A2: This is O(n) - it describes linear recursion where each step does constant work.

Q3: Why do we typically analyze worst-case performance?
A3: Worst-case gives an upper bound on running time, ensuring the algorithm will never perform worse than this. It's more reliable than best-case which may be unrealistic.

Questions & Answers

Q1: What's wrong with this code?

c
Copy
Download
char buffer[10];
scanf("%s", buffer);

A1: It's unsafe - no maximum width specified, risking buffer overflow. Should be scanf("%9s", buffer).

Q2: How do you properly compare two strings in C?
A2: Use strcmp() which returns:

  • Negative if first string comes first

  • Positive if first string comes after

  • 0 if equal

Q3: What's the difference between these declarations?

c
Copy
Download

char *b = "hello";

A3:

  • a is a mutable array on the stack

  • b is a pointer to immutable string literal in read-only memory

Questions & Answers

Q1: What are the potential issues with this code?

c
Copy
Download
int *arr = malloc(10 * sizeof(int));
// ... use arr ...
free(arr);
printf("%d", arr[0]);

A1: After freeing arr, accessing it is undefined behavior (dangling pointer). Should set arr = NULL after free.

Q2: Why use realloc instead of malloc+copy+free?
A2: realloc may be able to extend the existing block without copying, and handles the freeing of the old block.

Q3: What's the advantage of the doubling strategy for resizable arrays?
A3: It gives amortized O(1) time per append operation, even though some operations are O(n).

Questions & Answers

Q1: What are the advantages of using a wrapper structure for linked lists?
A1:

  1. Cleaner interface

  2. Can store additional metadata (like length)

  3. Reinforces proper usage patterns

  4. Easier to modify implementation

Q2: What's the time complexity of searching in a BST?
A2: O(h) where h is height. For balanced trees O(log n), for unbalanced could be O(n).

Q3: Why avoid node sharing in imperative languages?
A3: It can lead to:

  • Unexpected mutations when one list modifies shared nodes

  • Memory management issues (double frees or leaks)

Questions & Answers

Q1: How does qsort work with different data types?
A1: It uses:

  • void* pointers to data

  • size parameter for each element

  • Comparison function pointer that knows how to interpret the data

Q2: What are the issues with generic ADTs and memory management?
A2: Need clear protocol for:

  • Who allocates/frees items (ADT or client)

  • How to handle items that themselves contain dynamic memory

Q3: When would you choose an array vs linked list vs BST?
A3:

  • Array: Frequent random access, known maximum size

  • Linked list: Frequent insertions/deletions at ends

  • BST: Need efficient searching/sorting
