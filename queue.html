Queue in O(1)
Here's an implementation of a queue using a linked list in C with O(1) enqueue and dequeue operations. The key is to maintain pointers to both the front and rear of the queue for constant-time operations.
c
#include <stdio.h>
#include <stdlib.h>

// Node structure for the linked list
typedef struct Node {
    int data;
    struct Node* next;
} Node;

// Queue structure
typedef struct Queue {
    Node* front; // Points to the first node
    Node* rear;  // Points to the last node
} Queue;

// Initialize an empty queue
Queue* createQueue() {
    Queue* queue = (Queue*)malloc(sizeof(Queue));
    queue->front = NULL;
    queue->rear = NULL;
    return queue;
}

// Check if the queue is empty
int isEmpty(Queue* queue) {
    return queue->front == NULL;
}

// Enqueue: Add an element to the rear (O(1))
void enqueue(Queue* queue, int data) {
    Node* newNode = (Node*)malloc(sizeof(Node));
    newNode->data = data;
    newNode->next = NULL;
    
    if (isEmpty(queue)) {
        queue->front = newNode;
        queue->rear = newNode;
    } else {
        queue->rear->next = newNode;
        queue->rear = newNode;
    }
}

// Dequeue: Remove and return the front element (O(1))
int dequeue(Queue* queue) {
    if (isEmpty(queue)) {
        printf("Queue is empty\n");
        return -1; // Or handle error as needed
    }
    
    Node* temp = queue->front;
    int data = temp->data;
    queue->front = queue->front->next;
    
    if (queue->front == NULL) {
        queue->rear = NULL; // Queue is now empty
    }
    
    free(temp);
    return data;
}

// Peek: View the front element without removing it (O(1))
int peek(Queue* queue) {
    if (isEmpty(queue)) {
        printf("Queue is empty\n");
        return -1; // Or handle error as needed
    }
    return queue->front->data;
}

// Free the queue and all its nodes
void freeQueue(Queue* queue) {
    while (!isEmpty(queue)) {
        dequeue(queue);
    }
    free(queue);
}

// Example usage
int main() {
    Queue* queue = createQueue();
    
    enqueue(queue, 1);
    enqueue(queue, 2);
    enqueue(queue, 3);
    
    printf("Front element: %d\n", peek(queue)); // Output: 1
    printf("Dequeued: %d\n", dequeue(queue));   // Output: 1
    printf("Dequeued: %d\n", dequeue(queue));   // Output: 2
    printf("Front element: %d\n", peek(queue)); // Output: 3
    
    freeQueue(queue);
    return 0;
}
Explanation:
  • Structure:
    • Node represents a single element in the linked list with data and a pointer to the next node.
    • Queue maintains front (head) and rear (tail) pointers for O(1) access.
  • Enqueue (O(1)):
    • Creates a new node and adds it to the rear.
    • Updates rear to point to the new node.
    • If the queue is empty, sets both front and rear to the new node.
  • Dequeue (O(1)):
    • Removes the front node, returns its data, and updates front to the next node.
    • If the queue becomes empty, sets rear to NULL.
  • Peek (O(1)): Returns the front element without removing it.
  • Memory Management:
    • createQueue allocates memory for the queue.
    • freeQueue deallocates all nodes and the queue structure.
  • Error Handling: Checks for empty queue in dequeue and peek.
This implementation ensures both enqueue and dequeue operations are O(1) by directly accessing the front and rear pointers, avoiding traversal of the linked list.

Let's break down the else block in the enqueue function to clarify why both queue->rear->next = newNode and queue->rear = newNode are necessary:
c
else {
    queue->rear->next = newNode;
    queue->rear = newNode;
}
Context
  • The enqueue function adds a new node (with data) to the end of the queue, which is implemented as a singly linked list.
  • The Queue structure maintains two pointers:
    • front: Points to the first node (head) of the queue.
    • rear: Points to the last node (tail) of the queue.
  • The newNode is the newly created node to be added to the queue, with its next pointer set to NULL (since it will be the last node).
  • The else block executes when the queue is not empty, meaning there is at least one node already in the queue.
Why both assignments are needed
  1. queue->rear->next = newNode:
    • The rear pointer points to the current last node of the queue.
    • To add the newNode to the end of the queue, we need to link the current last node to the newNode.
    • Setting queue->rear->next = newNode updates the next pointer of the current last node to point to the newNode, effectively appending newNode to the linked list.
    • This maintains the linked list structure, ensuring the new node is connected to the existing chain.
  2. queue->rear = newNode:
    • After linking the newNode to the current last node, the newNode becomes the new last node of the queue.
    • To reflect this, we update the rear pointer to point to newNode.
    • This ensures that the rear pointer always points to the last node, which is critical for O(1) enqueue operations in future calls (since we can directly access the last node to append a new one).
Why both are necessary
  • The first assignment (queue->rear->next = newNode) connects the new node to the linked list, preserving the chain of nodes.
  • The second assignment (queue->rear = newNode) updates the rear pointer to reflect that the new node is now the last node in the queue.
  • If we only did queue->rear->next = newNode without updating queue->rear, the rear pointer would still point to the previous last node, and future enqueue operations would incorrectly append nodes after the old rear, breaking the queue structure.
  • If we only did queue->rear = newNode without setting queue->rear->next, the new node would not be linked to the existing list, effectively detaching the queue's previous nodes (causing a memory leak and breaking the queue).
Example WalkthroughSuppose the queue has two nodes: [1 -> 2], where front points to node 1, and rear points to node 2. The node 2 has next = NULL.
  • We want to enqueue a new node with data 3 (newNode).
  • In the else block:
    1. queue->rear->next = newNode: The next pointer of node 2 (the current rear) is set to point to newNode (node 3). Now the list is [1 -> 2 -> 3], with node 3 having next = NULL.
    2. queue->rear = newNode: The rear pointer is updated to point to node 3, the new last node.
  • Result: The queue is now [1 -> 2 -> 3], with front pointing to 1 and rear pointing to 3.
SummaryBoth assignments in the else block are essential:
  • queue->rear->next = newNode links the new node to the existing list.
  • queue->rear = newNode updates the rear pointer to maintain the queue's structure for O(1) operations.
This ensures the linked list remains properly connected and the rear pointer always points to the last node, enabling constant-time enqueue operations.
